# Техническое задание на разработку веб-панели Crypto Bot v5.x

## 1. Введение

### 1.1 Цель документа

Настоящее техническое задание определяет требования к разработке веб-панели управления и мониторинга для торговой системы Crypto Bot v5.x. Панель представляет собой одностраничное приложение (SPA), взаимодействующее с существующим FastAPI backend (`api/server.py`) через REST API и WebSocket.

### 1.2 Цели проекта

1. Визуальный мониторинг всех аспектов торговли в реальном времени
2. Управление стратегиями и параметрами риска без редактирования YAML
3. Конфигурирование TradingView webhook-алертов через UI
4. Просмотр аналитики с интерактивными графиками
5. Управление конфигурациями окружений (production/paper/backtest)
6. Быстрый запуск/остановка бота и отдельных стратегий

### 1.3 Основной принцип

Панель — это **view-слой** над существующим API. Она **не содержит** торговой логики. Все решения принимаются бэкендом. Панель только отображает состояние и передаёт команды пользователя.

## 2. Технологический стек

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| **Фреймворк** | React 19 + TypeScript | Стандарт индустрии, строгая типизация |
| **Сборка** | Vite | Быстрая сборка, HMR |
| **Стейт-менеджмент** | Zustand | Лёгкий, без boilerplate |
| **UI-компоненты** | Tailwind CSS + shadcn/ui | Современный дизайн, кастомизация |
| **Графики** | lightweight-charts (TradingView) | Профессиональные свечные графики |
| **HTTP-клиент** | axios + React Query (TanStack) | Кеширование, рефетч, optimistic updates |
| **WebSocket** | native WebSocket + React Query subscriptions | Real-time обновления |
| **Формы** | React Hook Form + Zod | Валидация на клиенте |
| **Роутинг** | React Router v7 | SPA-навигация |
| **Тестирование** | Vitest + Testing Library | Быстрые тесты |
| **Линтинг** | ESLint + Prettier | Качество кода |

## 4. Функциональные требования

### 4.1 Dashboard (Главный экран)

#### 4.1.1 Верхняя панель метрик

| Метрика | Источник | Обновление |
|---------|----------|------------|
| Баланс (USDT) | `GET /portfolio` | 5 сек |
| Эквити | `GET /portfolio` | 5 сек |
| Дневной PnL ($ / %) | `GET /portfolio` | 5 сек |
| Открыто позиций | `GET /portfolio` | 5 сек |
| Winrate | `GET /analytics/metrics` | 60 сек |
| Profit Factor | `GET /analytics/metrics` | 60 сек |
| Текущая просадка | `GET /portfolio` | 5 сек |
| Режим (Online/Recovery) | `GET /portfolio` | 5 сек |

Каждая метрика — компонент `MetricCard`:
- Значение крупным шрифтом
- Изменение за период (стрелка вверх/вниз, зелёный/красный)
- Sparkline-график за последние 24 часа

#### 4.1.2 График эквити

- Линейный график equity за выбранный период (1д / 1н / 1м / 3м / всё)
- Наложение: drawdown-область под графиком
- Точки входов/выходов на линии эквити

#### 4.1.3 Таблица открытых позиций

Колонки: Pair, Direction, Entry Price, Current Price, PnL ($), PnL (%), Stop Loss, TP1/TP2, Strategy, Age.

Действия:
- Кнопка «Закрыть позицию» (с подтверждением)
- Кнопка «Переместить стоп» (inline-редактирование)
- Клик по паре → переход на страницу позиции

#### 4.1.4 Последние сделки

Мини-таблица последних 10 сделок. Колонки: Time, Pair, Direction, PnL, Strategy. Клик → полная история.

#### 4.1.5 Статус системы

- Health-индикатор (зелёный/жёлтый/красный)
- WebSocket подключение (да/нет)
- Биржа + testnet/prod
- Загрузка CPU / память
- Время последнего цикла

### 4.2 Позиции (Positions Page)

#### 4.2.1 Полноценная таблица

- Сортировка по любой колонке
- Фильтр по паре, стратегии, направлению
- Массовые действия: закрыть все LONG / закрыть все SHORT
- Экспорт в CSV

#### 4.2.2 Детальная карточка позиции

При клике на позицию — модальное окно / сайд-панель:
- График цены с标注 входа, SL, TP1, TP2
- MAE/MFE в реальном времени
- История движения стопа
- Кнопки: закрыть, переместить стоп, переместить TP

### 4.3 Сделки (Trades Page)

#### 4.3.1 История сделок

- Пагинированная таблица (50/100/200 строк на странице)
- Фильтры: дата (календарь), пара, стратегия, результат (win/loss)
- Сортировка по дате, PnL, duration
- Цветовое кодирование: зелёный = win, красный = loss

#### 4.3.2 Статистика по фильтру

При применении фильтра автоматически пересчитываются:
- Количество сделок
- Winrate
- Суммарный PnL
- Средний PnL на сделку
- Profit Factor

#### 4.3.3 График распределения

- Гистограмма PnL (распределение прибылей/убытков)
- Круговая диаграмма win/loss
- График PnL по дням недели / часам

### 4.4 Стратегии (Strategies Page)

#### 4.4.1 Список стратегий

Для каждой стратегии (Sweep / Bounce / Breakout):
- Статус (вкл/выкл)
- Текущие параметры (wick_ratio, volume_multiplier, min_rr, …)
- Количество сделок, Winrate, PnL, PF
- Кнопка включения/выключения

#### 4.4.2 Редактор стратегии

При клике на стратегию — форма редактирования:
- Все параметры из `StrategyConfig`
- Валидация (min ≤ max, положительные значения, …)
- Кнопка «Сохранить» → `PUT /api/strategies/:name`
- Кнопка «Сбросить к значениям из конфига»

#### 4.4.3 Детали стратегии

Отдельная страница `/strategies/:name`:
- График эквити только по этой стратегии
- Winrate по месяцам (столбчатая диаграмма)
- Таблица сделок только этой стратегии
- Bayesian winrate (текущий + credible interval)
- Среднее время удержания позиции
- Распределение по рыночным режимам (в каком режиме стратегия зарабатывает)

#### 4.4.4 Управление плагинами

Список плагинов (из `plugins/loader.py`):
- Загруженные плагины с иконкой статуса
- Кнопка «Перезагрузить плагины»
- Возможность загрузить новый `.py` файл

### 4.5 Риск (Risk Page)

#### 4.5.1 Текущие параметры риска

Форма со всеми параметрами из `risk_engine`:
- `max_risk_per_trade`: слайдер 0.1% – 5.0%
- `max_positions`: 1 – 10
- `max_correlation`: 0.1 – 1.0
- `max_exposure`: 1.0% – 10.0%
- Stop multipliers для каждого режима
- Drawdown limits (daily/weekly/monthly/total)

#### 4.5.2 Визуализация лимитов

- Горизонтальные progress-бары:
  - Дневная просадка / лимит
  - Недельная просадка / лимит
  - Месячная просадка / лимит
  - Общая просадка / лимит
- Цвет: зелёный (< 50% лимита) → жёлтый → красный (> 80%)

#### 4.5.3 Recovery Mode

Блок статуса Recovery Mode:
- Активен? (да/нет)
- Текущая просадка
- Порог входа / выхода
- Количество последовательных побед
- Требуется для выхода
- Кнопка «Принудительный выход из Recovery» (с подтверждением)

### 4.6 Аналитика (Analytics Page)

#### 4.6.1 Сводные метрики

Все метрики из `GET /analytics/metrics`:
- Winrate, Profit Factor, Expectancy
- Sharpe Ratio, Calmar Ratio, Recovery Factor
- Средний win / средний loss
- Комиссии всего
- Среднее проскальзывание

#### 4.6.2 Графики

- **Equity Curve** — кумулятивный PnL
- **Drawdown** — под эквити
- **Monthly Returns** — столбцы по месяцам (зелёные/красные)
- **Rolling Sharpe** (30-дневное окно)
- **Winrate by Weekday** — столбчатая диаграмма
- **PnL Distribution** — гистограмма

Все графики с выбором периода и возможностью зума.

#### 4.6.3 Дневной отчёт

Контент из `GET /analytics/daily`. Отображается:
- Сделок сегодня
- PnL сегодня
- Winrate сегодня
- Лучшая / худшая сделка

#### 4.6.4 Бэктестинг

Страница `/analytics/backtest`:
- Выбор конфигурации (`backtest.yaml`)
- Выбор периода (date picker: start / end)
- Выбор пар
- Кнопка «Запустить бэктест» → `POST /api/backtest/run`
- Прогресс-бар выполнения
- Результаты: метрики, эквити-график, сделки

### 4.7 TradingView (TradingView Page)

#### 4.7.1 Настройка Webhook

- Webhook URL (копируется по клику)
- Статус webhook (активен/неактивен)
- Webhook token (сгенерировать / показать / скрыть)
- Тестовый запрос (кнопка «Отправить тестовый алерт»)

#### 4.7.2 Индикаторы

Таблица индикаторов из `GET /webhook/indicators`:
- Название
- Параметры
- Кнопка «Скопировать PineScript» (копирует шаблон в буфер)
- Кнопка «Настроить алерт» → форма с предзаполненными полями

#### 4.7.3 Социальные сигналы

- Fear & Greed Index — крупно, с цветовой шкалой
- График F&G за 30 дней
- Социальный объём (график)
- Активность китов (таблица)
- Sentiment по выбранной паре

#### 4.7.4 История алертов

Таблица из `GET /webhook/alerts/recent`:
- Время, Action, Symbol, Status (processed/rejected/error), Signal Created
- Фильтр по символу, статусу
- Автообновление каждые 10 секунд

### 4.8 Конфигурация (Config Page)

#### 4.8.1 Выбор окружения

Три вкладки: Production / Paper / Backtest.

Для каждого окружения:
- Название файла
- Дата последнего изменения
- Кнопка «Активировать»
- Кнопка «Редактировать»

#### 4.8.2 Редактор YAML

Два режима:
1. **Form mode** — структурированная форма с группировкой по секциям
2. **Raw mode** — Monaco Editor (подсветка YAML, валидация)

Кнопка «Сохранить» → `PUT /api/config/:env`. Кнопка «Валидировать» (проверка без сохранения).

#### 4.8.3 История версий

Таблица из `GET /config/versions`:
- Версия, дата, хеш
- Кнопка «Откатить» (с подтверждением)
- Diff между версиями (подсветка изменений)

#### 4.9.3 Uptime

#### 4.9.4 Логи

Страница `/monitor/logs`:
- Поток логов в реальном времени (WebSocket или SSE)
- Фильтр по уровню (DEBUG / INFO / WARNING / ERROR / CRITICAL)
- Фильтр по сервису (data_service, strategy_engine, …)

- Авто-скролл (вкл/выкл)

## 5. API-контракт (новые эндпоинты)

Существующие эндпоинты (GET /health, /portfolio, …) остаются без изменений. Добавляются:

### 5.1 Управление стратегиями

```
GET    /api/strategies              → список стратегий с метриками
GET    /api/strategies/:name        → детали + метрики
PUT    /api/strategies/:name        → обновить параметры
POST   /api/strategies/:name/toggle → вкл/выкл
```

### 5.2 Управление риском

```
GET    /api/risk                    → текущие параметры риска
PUT    /api/risk                    → обновить параметры
POST   /api/risk/recovery/exit      → принудительный выход из Recovery
```

### 5.3 Бэктестинг

```
POST   /api/backtest/run            → запустить бэктест
GET    /api/backtest/status/:id     → статус бэктеста
GET    /api/backtest/results/:id    → результаты
GET    /api/backtest/history        → история бэктестов
```

### 5.4 Логи

```
GET    /api/logs                    → последние N строк логов
WS     /ws/logs                     → real-time stream логов
```

### 5.5 WebSocket-события

```
WS     /ws/events                   → все события из EventBus
```

Формат сообщений:
```json
{
  "topic": "signal.generated",
  "data": {"pair": "BTCUSDT", "direction": "LONG", "confidence": 0.85},
  "timestamp": "2026-07-14T10:30:00Z"
}
```

Топики:
- `signal.generated`, `signal.approved`, `signal.rejected`
- `order.placed`, `order.filled`, `order.failed`
- `position.opened`, `position.closed`, `position.stop_moved`
- `regime.changed`, `recovery.entered`, `recovery.exited`
- `health.changed`
- `tv.alert_received`, `tv.signal_created`

### 5.6 Управление ботом

```
POST   /api/bot/start               → запустить бота
POST   /api/bot/stop                → остановить бота
POST   /api/bot/restart             → перезапустить
GET    /api/bot/status              → статус (running/stopped), uptime, version
```

### 5.7 Настройки

```
GET    /api/settings                → все настройки
PUT    /api/settings                → обновить настройки
POST   /api/settings/exchange/test  → проверить подключение к бирже
GET    /api/settings/notifications  → настройки уведомлений
PUT    /api/settings/notifications  → обновить уведомления
```

## 6. Нефункциональные требования

### 6.1 Производительность

- Первая отрисовка (LCP): < 2 сек
- Интерактивность (FID): < 100 мс
- Обновление real-time данных: < 500 мс от события до отображения
- 60 fps при скролле таблиц (виртуализация для 10 000+ строк)
- Бандл: < 500 KB (gzip)

### 6.2 Надёжность

- Автоматический реконнект WebSocket при разрыве
- Грациозная деградация при недоступности API (показ закешированных данных)
- Офлайн-режим: показ последнего известного состояния с индикатором «нет связи»
- Retry с exponential backoff для REST-запросов

### 6.3 Безопасность

- JWT аутентификация (логин/пароль) на бэкенде
- Все запросы к API с токеном в заголовке
- Автоматический logout при истечении токена
- HTTPS only (в production)
- CSP-заголовки для защиты от XSS

### 6.4 Доступность

- Поддержка светлой и тёмной темы
- Адаптивная вёрстка: десктоп (1920px), планшет (1024px), мобильный (375px)
- Клавиатурная навигация
- Screen-reader friendly (aria-метки)

### 6.5 Масштабируемость

- Ленивая загрузка страниц (code splitting по роутам)
- Кеширование API-ответов на клиенте (TanStack Query)
- Виртуализация длинных списков

## 10. Риски и смягчение

| Риск | Вероятность | Смягчение |
|------|------------|-----------|
| CORS-ошибки между SPA и API | Средняя | Прокси через Nginx, CORS middleware на FastAPI |
| WebSocket нестабилен | Средняя | Авто-реконнект + fallback на polling |
| Большой объём данных в таблицах | Средняя | Виртуализация (TanStack Virtual) |
| Расхождение типов front/back | Низкая | Генерация TypeScript-типов из OpenAPI-схемы |
| Утечки памяти в графиках | Низкая | Правильный unmount, очистка подписок |
